Как пользоваться гитом

Глоссарий

  • Гит (git) — система управления версиями проекта.
  • Репозиторий (repo) — каталог, в котором хранятся файлы проекта. Может быть локальным или удаленным. Файлы локального репозитория хранятся на машине пользователя, в то время как удаленный репозиторий размещается на серверах GitHub'а или любого другого поддерживающего гит хостинга проектов.
  • Ветка (branch) — набор последовательно сохраненных версий проекта.
  • Коммит (commit) — загрузка снимка репозитория на сервер.

Как сделать коммит

  1. Перейти в каталог локального репозитория;
  2. Обновить локальный репозиторий командой git reset --hard origin/<название ветки>, чтобы его версия соответствовала версии удаленного репозитория;
  3. Сделать требуемые изменения;
  4. Перейти в рабочую ветку командой git ckeckout <название ветки>;;
  5. Добавить нужные файлы в коммит командой git add <имя файла>;
  6. Сформировать коммит командой git commit -m <комментарий>;
  7. Загрузить изменения в удаленный репозиторий командой git push -u origin <название ветки>.

Прочие полезные команды

КОМАНДА ОПИСАНИЕ
git status Просмотр сведений о состоянии файлов, в т. ч. сравнение их версий в локальном и удаленном репозиториях.
